Here’s a fully converted PostgreSQL version of your Docker Compose setup. 
Preserved your volume structure, Mautic mounts, and Strapi integration while switching the database from MySQL → PostgreSQL.

- IF YOU ARE TRYING TO USE PostgreSQL:
- you may have exactly the follow-up issue:

The "MYSQL_HOST" variable is not set. Defaulting to a blank string.
dependency failed to start: container soloai-dev-mautic_web-1 exited (1)

Here’s the problem:
- Mautic currently only supports MySQL/MariaDB. It cannot connect to PostgreSQL out-of-the-box.
- Your .env now only has POSTGRES_* variables, but mautic_web container is still trying to read MYSQL_* variables.
- Docker Compose is warning that all MYSQL_* vars are missing → the container fails immediately.

The solution
- You have two options:

Option 1 (Recommended): Keep MySQL for Mautic
- Mautic works with MySQL/MariaDB only. You can still use PostgreSQL for Strapi/Prisma.
1- Restore the MySQL env vars in your .env:

MYSQL_HOST=db
MYSQL_PORT=3306
MYSQL_DATABASE=mautic_db
MYSQL_USER=mautic_db_user
MYSQL_PASSWORD=superSecretUserPwd
MYSQL_ROOT_PASSWORD=superSecretRootPwd

2- Make your db service MySQL:

db:
  image: mysql:8.0
  environment:
    - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
    - MYSQL_USER=${MYSQL_USER}
    - MYSQL_PASSWORD=${MYSQL_PASSWORD}
    - MYSQL_DATABASE=${MYSQL_DATABASE}

3- Keep a separate PostgreSQL container for Strapi/Prisma if needed.
- This will immediately fix the Mautic container crash.


Option 2 (Not Recommended): PostgreSQL-only setup
If you insist on PostgreSQL for Mautic:
- You’d need to apply a custom Mautic patch or use a third-party DB driver, because the official Docker image does not support Postgres.
- This requires advanced hacking: patching PDO drivers in the container and changing mautic_env. It’s not trivial.

Bottom line: For production or MVP, use MySQL for Mautic.

Additional Notes
- DOCKER_MAUTIC_LOAD_TEST_DATA and DOCKER_MAUTIC_RUN_MIGRATIONS warnings are harmless; you can define them in .env if you want to suppress warnings:

DOCKER_MAUTIC_LOAD_TEST_DATA=0
DOCKER_MAUTIC_RUN_MIGRATIONS=1

Your Strapi/Postgres setup is fine; it’s only Mautic that requires MySQL.

If you are using PostgreSQL copy/paste the following to your docker-compose.yml:

x-soloai-volumes:
  &soloai-volumes
  - ${BASE_PATH}/mautic/config:/var/www/html/config:z
  - ${BASE_PATH}/mautic/logs:/var/www/html/var/logs:z
  - ${BASE_PATH}/mautic/media/files:/var/www/html/docroot/media/files:z
  - ${BASE_PATH}/mautic/media/images:/var/www/html/docroot/media/images:z
  - ${BASE_PATH}/cron:/opt/mautic/cron:z
  - mautic-docroot:/var/www/html/docroot:z

services:
  db:
    image: postgres:16
    environment:
      - POSTGRES_DB=${POSTGRES_DB}
      - POSTGRES_USER=${POSTGRES_USER}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    ports:
      - "5432:5432"
    volumes: 
      - ${BASE_PATH}/postgres-data:/var/lib/postgresql/data
      - ./init-strapi-db.sql:/docker-entrypoint-initdb.d/init-strapi-db.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
      start_period: 5s
      interval: 5s
      timeout: 5s
      retries: 10
    networks:
      - default

  nginx:
    image: nginx
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
      - mautic-docroot:/var/www/html/docroot:z
      - ./mautic/media/files:/var/www/html/docroot/media/files:z
      - ./mautic/media/images:/var/www/html/docroot/media/images:z
    depends_on:
      - mautic_web
    ports:
      - 8080:80
    networks:
       - default
  
  mautic_web:
    image: mautic/mautic:6-fpm
    links:
      - db:postgres
    volumes: *soloai-volumes
    environment:
      - DOCKER_MAUTIC_LOAD_TEST_DATA=${DOCKER_MAUTIC_LOAD_TEST_DATA}
      - DOCKER_MAUTIC_RUN_MIGRATIONS=${DOCKER_MAUTIC_RUN_MIGRATIONS}
    env_file:
      - .mautic_env
    healthcheck:
      test: cgi-fcgi -bind -connect 127.0.0.1:9000
      start_period: 5s
      interval: 5s
      timeout: 5s
      retries: 100
    depends_on:
      db:
        condition: service_healthy
    networks:
      - default

  mautic_cron:
    image: mautic/mautic:6-fpm
    links:
      - db:postgres
    volumes: *soloai-volumes
    environment:
      - DOCKER_MAUTIC_ROLE=mautic_cron
    env_file:
      - .mautic_env
    depends_on:
      mautic_web:
        condition: service_healthy
    networks:
      - default

  mautic_worker:
    image: mautic/mautic:6-fpm
    links:
      - db:postgres
    volumes: *soloai-volumes
    environment:
      - DOCKER_MAUTIC_ROLE=mautic_worker
    env_file:
      - .mautic_env
    depends_on:
      mautic_web:
        condition: service_healthy
    networks:
      - default

  strapi:
    image: vshadbolt/strapi
    environment:
      NODE_ENV: development
      DATABASE_CLIENT: postgres
      DATABASE_HOST: ${POSTGRES_HOST}
      DATABASE_PORT: ${POSTGRES_PORT}
      DATABASE_NAME: strapi_db
      DATABASE_USERNAME: ${POSTGRES_STRAPI_USER}
      DATABASE_PASSWORD: ${POSTGRES_STRAPI_PASSWORD}
      JWT_SECRET: ${JWT_SECRET}
      ADMIN_JWT_SECRET: ${ADMIN_JWT_SECRET}
      APP_KEYS: ${APP_KEYS}
      TRANSFER_TOKEN_SALT: ${TRANSFER_TOKEN_SALT}
      API_TOKEN_SALT: ${API_TOKEN_SALT}
      DATABASE_SSL: false
    ports:
      - "1337:1337"
    depends_on:
      db:
        condition: service_healthy
    volumes:
      - strapi-app:/srv/app:z
    networks:
      - default      

volumes:
  postgres-data:
    driver: local
    driver_opts:
      type: none
      device: ${BASE_PATH}/postgres-data
      o: bind
  mautic-docroot:
    driver: local
    driver_opts:
      type: none
      device: ${BASE_PATH}/mautic-docroot
      o: bind
  strapi-app:
    driver: local
    driver_opts:
      type: none
      device: ${BASE_PATH}/strapi
      o: bind

networks:
  default:
    name: ${COMPOSE_PROJECT_NAME}-docker
